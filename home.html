This is the Introduction to the process flow for the application. Other items, such as user preferences and performance characteristics for various projects and developers on those projects are not covered here as they are more advanced topics.

1. Create a new project.

2. Add tasks to your new project.

3. Add a team. While not essential to add members it gives those folks some accountability for the tasks given and the outcome of the project.

4. Add tasks from your project to the sprint so that people can see and work on that particular task.

5. You the sprint activity page. This keeps track of what 'stage' a task is in during development. You can add questions or comments, which will show up on the home page for the other users.
